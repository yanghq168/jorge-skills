# 安全应急响应工作流

本skill定义了安全事件应急响应的完整多Agent协作流程，通过YAML编排实现各Agent的依赖管理和数据传递。

## 工作流概览

```
                    ┌→ 通知沟通Agent ──┐
事件发现 → 安全分析Agent ─┼→ 漏洞修复Agent ──┼→ 事件闭环Agent
                    └→ 取证调查Agent ──┘
```

## 关键特性

- **紧急响应**: 支持高优先级快速响应，跳过非关键确认点
- **并行处理**: 通知、修复、调查可并行执行
- **自动升级**: 严重事件自动升级至更高级别处理
- **完整追溯**: 全程记录事件处理时间线和决策依据

## 使用方式

1. 用户报告安全事件或系统检测到异常
2. 指挥官（权权管家）读取本skill中的YAML工作流定义
3. 立即启动安全分析Agent进行初步评估
4. 根据严重级别决定后续Agent的启动策略
5. 并行启动通知、修复、调查Agent
6. 指挥官监控各Agent进度，协调资源
7. 事件闭环Agent完成总结和复盘

## 快速示例

**用户输入**: "服务器被入侵了，有大量异常流量"

**指挥官执行流程**:
1. 立即启动安全分析Agent评估事件严重级别
2. 判定为P1严重事件，并行启动：
   - 通知沟通Agent：通知安全团队和管理层
   - 漏洞修复Agent：隔离受影响系统、阻断攻击
   - 取证调查Agent：收集日志、追踪攻击路径
3. 指挥官实时同步各Agent进展
4. 修复完成后启动事件闭环Agent
5. 输出完整事件报告和改进建议

## 严重级别定义

| 级别 | 描述 | 响应时间 | 升级条件 |
|------|------|----------|----------|
| P0 | 灾难级 | 5分钟 | 立即全员通知 |
| P1 | 严重 | 15分钟 | 管理层+技术负责人 |
| P2 | 一般 | 1小时 | 相关团队通知 |
| P3 | 轻微 | 4小时 | 邮件通知 |

## 关键原则

- **时间优先**: 安全事件响应速度优先于完美
- **并行处理**: 通知、修复、调查三线并行
- **最小影响**: 修复措施优先保证业务连续性
- **完整记录**: 所有操作和时间节点完整记录
