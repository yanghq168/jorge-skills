# 网站宣传策略工作流

本skill定义了基于网站内容自动生成宣传策略的完整多Agent协作流程。

## 工作流概览

```
网站分析Agent → 竞品分析Agent → 宣传策略Agent → 内容创作Agent → 渠道规划Agent
      ↓              ↓              ↓              ↓              ↓
  网站定位分析   竞品策略洞察   宣传方案制定   宣传素材创作   渠道投放计划
```

## 使用方式

1. 用户提供目标网站URL
2. 指挥官（权权管家）读取本skill中的YAML工作流定义
3. 启动网站分析Agent爬取和分析网站内容
4. 基于分析结果启动竞品分析Agent
5. 宣传策略Agent综合信息制定策略
6. 内容创作Agent生成宣传素材
7. 渠道规划Agent制定投放计划
8. 指挥官整合输出完整宣传方案

## 工作流文件

- [workflow.yaml](references/workflow.yaml) - 完整YAML工作流定义
- [agent-roles.md](references/agent-roles.md) - 各Agent角色职责说明

## 快速示例

**用户输入**: "帮我为 https://example.com 制定一个宣传策略"

**指挥官执行流程**:
1. 网站分析Agent：爬取网站，分析产品定位、目标受众、核心卖点
2. 竞品分析Agent：分析同类产品的宣传策略和差异化机会
3. 宣传策略Agent：制定定位策略、核心信息、传播角度
4. 内容创作Agent：创作宣传文案、海报概念、视频脚本
5. 渠道规划Agent：制定渠道组合和投放计划
6. 输出完整宣传策略方案

## 关键原则

- **数据驱动**: 基于网站实际内容和竞品数据制定策略
- **差异化**: 强调与竞品的差异化优势
- **可执行**: 策略必须配套具体的内容和渠道计划
- **用户确认**: 策略方向确定后需用户确认再生成素材
