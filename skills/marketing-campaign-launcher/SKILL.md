# 营销活动启动工作流

本skill定义了营销活动启动的完整多Agent协作流程，通过YAML编排实现各Agent的依赖管理和数据传递。

## 工作流概览

```
需求分析Agent → 策略制定Agent → 内容创作Agent → 渠道分发Agent → 数据监控Agent
     ↓              ↓              ↓              ↓              ↓
  输出需求文档   输出策略方案   输出素材包     输出发布计划    输出监控报告
```

## 使用方式

1. 用户提出营销活动需求
2. 指挥官（权权管家）读取本skill中的YAML工作流定义
3. 按顺序调度各Agent执行任务
4. 各Agent通过共享上下文传递数据
5. 指挥官整合各Agent输出，形成完整营销方案

## 工作流文件

- [workflow.yaml](references/workflow.yaml) - 完整YAML工作流定义
- [agent-roles.md](references/agent-roles.md) - 各Agent角色职责说明

## 快速示例

**用户输入**: "帮我策划一场新品发布会营销活动"

**指挥官执行流程**:
1. 读取 workflow.yaml 了解完整流程
2. 调用需求分析Agent收集产品信息、目标受众、预算
3. 调用策略制定Agent基于需求输出营销策略
4. 调用内容创作Agent生成宣传文案、海报脚本
5. 调用渠道分发Agent制定投放计划
6. 调用数据监控Agent设计追踪指标
7. 整合输出完整营销方案

## 关键原则

- **依赖管理**: 每个Agent等待前置Agent完成后才启动
- **数据传递**: 通过结构化上下文在各Agent间传递数据
- **异常处理**: 任一Agent失败时，指挥官决定是否重试或跳过
- **用户确认**: 关键节点（策略确定后）需用户确认后再继续
